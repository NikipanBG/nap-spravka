# НАП · Справка + OCR на фактури

Уеб приложение (един HTML файл) за работа с декларирани продажби по ЗДДС:
**снимай фактура → извлечи данните автоматично → сравни с справката от НАП.**

Всичко работи изцяло в браузъра. Нищо не се качва на сървър — данните се пазят само на твоето устройство (localStorage).

---

## Какво прави

Четири таба:

1. **📷 OCR Фактури** — качваш снимка/PDF на фактура, изкуственият интелект (Claude) извлича дата, номер, издател, ЕИК/ДДС №, основа и ДДС. Прегледаш и натискаш „Добави към Файл Б".
2. **📥 Файл А · НАП** — зареждаш `.xlsx` справката, изтеглена от портала на НАП.
3. **✏️ Файл Б · Въвеждане** — твоите фактури (попълнени ръчно, импортирани или дошли от OCR). Автоматично маркира дублирани номера. Има архив за резервно копие.
4. **⚡ Сравнение** — сравнява А срещу Б и показва разлики, липсващи и неплатени документи. Експорт в CSV/XLSX.

## Качване на снимки — телефон и Google Drive

В таб **OCR Фактури** има три бутона:

- **📷 Снимай** — отваря камерата на телефона (или галерията).
- **📁 Файл** — отваря системния избор. **На телефон тук вече се показва Google Drive, Снимки, Файлове** — без никаква настройка.
- **☁ Drive** — директен избор от Google Drive на компютъра. Изисква еднократна настройка (виж по-долу).

> 💡 На телефон най-лесно е: **📷 Снимай** за нова снимка, или **📁 Файл → Google Drive** за вече качена снимка.

### Настройка на бутона ☁ Drive (само за компютър, по желание)

1. Отвори [console.cloud.google.com](https://console.cloud.google.com) и създай проект.
2. **APIs & Services → Library** → включи **Google Drive API** и **Google Picker API**.
3. **Credentials → Create credentials → API key** → копирай го.
4. **Credentials → Create credentials → OAuth client ID** → тип *Web application*.
5. В *Authorized JavaScript origins* добави адреса на сайта (напр. `https://потребител.github.io`).
6. Копирай Client ID.
7. В приложението: бутон **⚙ Google Drive** (горе вдясно) → постави Client ID и API Key → Запази.

Идентификаторите се пазят само в твоя браузър.

## Anthropic API ключ (за OCR)

OCR-ът използва Claude. Нужен е ключ от [console.anthropic.com](https://console.anthropic.com/settings/keys).
Въвеждаш го веднъж в таб **OCR Фактури**; пази се само локално в браузъра ти.

## Качване в GitHub и пускане онлайн (GitHub Pages)

За да го отваряш от телефона, качи го като GitHub Pages сайт:

1. Влез в [github.com](https://github.com) → **New repository** (напр. име `nap-spravka`), направи го **Public**.
2. Качи файла `index.html` (бутон **Add file → Upload files** → пусни `index.html` → **Commit changes**).
3. В хранилището: **Settings → Pages**.
4. Под *Build and deployment → Source* избери **Deploy from a branch**, branch **main**, папка **/ (root)** → **Save**.
5. След минута-две сайтът е достъпен на:
   `https://ПОТРЕБИТЕЛ.github.io/nap-spravka/`
6. Отвори този адрес на телефона и го добави към началния екран за бърз достъп.

> Този адрес е и това, което слагаш в *Authorized JavaScript origins* за Google Drive (само `https://ПОТРЕБИТЕЛ.github.io`).

## Поверителност

- Фактурите за OCR се изпращат към Anthropic API само за разпознаване.
- Справките от НАП, Файл Б и извлечените редове **не напускат браузъра** — стоят в localStorage.
- Бутонът **🗑 Нулирай** трие всичко локално.
- Прави редовно **архив** (таб Файл Б) — localStorage може да се изчисти при почистване на браузъра.
